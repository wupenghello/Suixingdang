## §0 使用说明（引用机制）

1. **任何 UI 改动的 PR 必须注明条款号**，格式：`依据 UI_SPEC §5.1 卡片·选中态`。无条款依据的视觉发明一律打回。
2. 规范没覆盖的场景：先按 §1 四条原则裁决，落地后**回补条款**（规范是活的，但修订必须走 PR）。
3. 改 `app.css` / `app.js` / `tokens.css` 必须升 `index.html` 中的 `?v=` 版本号（缓存铁律）。
4. 每页改造完成后按 §8 验收硬指标逐项扫描（8898 真实页面 + 全元素尺寸扫描，不许只看改动区域）。

## §1 四条设计原则（冲突时的最终裁决依据）

| 编号 | 原则 | 裁决示例 |
|---|---|---|
| P1 | **结构用灰，语义用色**：灰阶承担 90% 的层级表达；彩色只出现在「动作」与「状态反馈」两类场景 | 置顶是正向标记不是状态反馈 → 禁用橙色（§3.2） |
| P2 | **同一组件只有一种长相**：卡片、按钮、菜单、空状态全站一套解剖结构，页面间不许各画各的 | 文件卡与笔记卡共用骨架，仅内容区不同（§9） |
| P3 | **每个状态都必须被设计**：默认只是六分之一——悬停、按下、聚焦、选中、加载、失败缺一不可 | 没有失败态的列表不许上线（§5.1 / §5.6） |
| P4 | **像素落在 4 的倍数上**：间距、高度、内边距全部 4px 栅格；色值全部出自令牌 | 现有 18px 内边距、11px 字号均属违规（§2.1 / §10） |

## §2 令牌体系（`tokens.css` 待增补四族，色板族不动）

### §2.1 间距族（4px 栅格，强制）

| 令牌 | 值 | 用途 |
|---|---|---|
| `--space-1` | 4px | 图标与文字、胶囊内上下 |
| `--space-2` | 8px | 卡片内区块间隙、按钮组间距 |
| `--space-3` | 12px | 栅格间隙（≥1440px 升 16） |
| `--space-4` | 16px | 卡片内边距、区块内间距 |
| `--space-5` | 20px | 区块间距 |
| `--space-6` | 24px | 页面留白、大区块间距 |
| `--space-7` | 28px | — |
| `--space-9` | 36px | 分区间距 |
| `--space-12` | 48px | 编辑器内容上下留白 |

### §2.2 字阶族（Geist，六档封顶，禁止档外自创）

| 令牌 | 字号/字重/字距/行高 | 唯一用途 |
|---|---|---|
| `--text-page` | 20 / 600 / -0.01em / 28 | 页面标题（笔记、文件库、回收站…） |
| `--text-card-title` | 15 / 600 / -0.01em / 22 | 卡片标题、编辑器标题输入 |
| `--text-body` | 14 / 400 / — / 22 | 正文、菜单项 |
| `--text-secondary` | 13 / 400 / — / 20 | 摘要、说明文字 |
| `--text-meta` | 12 / 400 / — / 16，色 `--text-muted` | 时间、大小、字数、引用数。**永远 sans 字体，永不 mono** |
| `--text-badge` | 11 / 500 / — / 16 | 徽章、角标、`+N` 溢出 |

> 等宽字体（`--font-mono`）仅允许出现在：代码块、密钥/令牌值、纯数字统计表格。

### §2.3 高度族（控件统一）

| 场景 | 高度 |
|---|---|
| 页头 | 56 |
| 工具栏 / 筛选栏 | 40 |
| 编辑器顶栏 | 48 |
| 输入框 / 主按钮 / 次按钮 | 32 |
| 标签胶囊 / 分段开关 | 28 |
| 图标按钮 | 28 × 28 |
| 触控目标 | ≥ 44（由外边距补足，视觉尺寸可小于 44） |

### §2.4 圆角族（现状已有，锁定用法）

| 令牌 | 值 | 用法 |
|---|---|---|
| `--radius-sm` | 6px | 图标按钮、小徽章 |
| `--radius` | 8px | 输入框、按钮、菜单 |
| `--radius-lg` | 10px | **卡片**、弹层内面板 |
| `--radius-pill` | 999px | 胶囊 |

### §2.5 层级族（投影 × 边框分工铁律）

| 层级 | 表达 | 场景 |
|---|---|---|
| L0 静置 | 仅 `1px var(--border)`，无投影 | 卡片、输入框默认 |
| L1 交互 | `--shadow-sm` + 边框升 `--border-strong` | 悬停 |
| L2 浮起 | `--shadow` | 菜单、气泡、悬停操作条、Toast |
| L3 覆盖 | `--shadow-lg` | 编辑器全屏层、模态 |

> 铁律：**浅色模式靠边框分层，投影只在交互时出现**。卡片悬停投影用 `--shadow-sm`，禁止静置卡片带投影。

### §2.6 动效族

| 令牌 | 值 | 场景 |
|---|---|---|
| `--dur-fast` | 120ms ease-out | 按下、操作条/清除钮淡入淡出 |
| `--dur-base` | 160ms cubic-bezier(.2,0,0,1) | 悬停三件套（边框+投影+位移）、菜单弹出 |
| `--dur-slow` | 240ms ease-out | 编辑器开合、面板展开、视图切换 |
| 骨架微光 | 1.4s ease-in-out 循环 | 加载占位 |

> `prefers-reduced-motion: reduce` 时全部动画静止（现有 `app.css:2186` 模式推广全站）。

## §3 颜色用法矩阵

### §3.1 六类角色（全页面色彩裁决唯一依据）

| 角色 | 令牌 | 唯一合法场景 | 禁区 |
|---|---|---|---|
| 品牌动作 | `--primary` 族 | 主按钮、链接、激活导航、选中态、焦点环、强调胶囊 | 禁止纯装饰、禁止表达状态 |
| 成功 | `--success` 族 | 保存成功、操作完成反馈 | 禁止用于置顶这类正向标记 |
| 警告 | `--warning` 族 | **未保存**、配额将满、功能降级 | 禁止用于置顶徽章 |
| 危险 | `--danger` 族 | 破坏性操作悬停、错误、失败态 | **禁止默认出现在顶栏**（降级进溢出菜单后可在菜单内悬停出现） |
| 身份 | `--type-*` | 仅文件类型图标着色 | 禁止做文字色 / 背景色 |
| 中性结构 | 灰阶族 | 布局、边框、层级、次要信息 | — |

### §3.2 首批修正三例（笔记页落地时强制执行）

| 元素 | ✕ 现状 | ✓ 标准 |
|---|---|---|
| 置顶徽章 | `--warning` 橙底 chip（`app.css:2028`） | 标题左侧 14px 品牌蓝图钉图标 +「置顶」分区承载权重，卡片不染色 |
| AI 标签胶囊 | 灰底灰字（`app.css:2084`，与背景糊在一起） | `--primary-light` 底 + **虚线边框** + `AI` 前缀 |
| 未保存徽标 | 不存在 | `--warning-light` 底 + 橙点「未保存」——警告色第一次用在它该用的地方（§5.7） |

### §3.3 铁律

- 全文件**零孤立色值**：任何 `#xxx` / `rgba(...)` 字面量出现在 `app.css` 即 CI 判负（现状违规点见 §10 审计 #1/#6）。
- 焦点环统一 `2px solid var(--primary-ring)` + `outline-offset: 2px`。

## §4 页面骨架规范（所有列表页共用模板）

### §4.1 结构

```
┌─ 页头 56px ────────────────────────────────────────────────┐
│ 页面标题(20/600)  计数(13/muted)      [搜索][排序▾][⊞/☰][+ 主按钮] │  ← 主按钮恒在最右
├─ 工具栏 40px（滚过页头后吸顶）───────────────────────────────┤
│ [全部][#标签 计数]…                          已筛选 N 篇 · 清除  │
├─ 内容区 padding 24（移动端 16）──────────────────────────────┤
│ 分区标题(13/600/muted) + 计数                                 │
│ 卡片栅格 repeat(auto-fill, minmax(240px,1fr))，gap 12         │
├─ 哨兵区 ──────────────────────────────────────────────────┤
│ 触底：三格骨架「加载中…」 / 到底：「已显示全部 N 篇」            │
└────────────────────────────────────────────────────────────┘
```

### §4.2 响应式断点

| 断点 | 行为 |
|---|---|
| ≥1440px | 栅格 gap 升 16px |
| 1024–1439 | 基准 |
| 768–1023 | 工具栏搜索收窄至 180px |
| <768px | 栅格单列；页面留白 16px；工具栏换行；卡片操作按钮常显（无悬停概念） |

### §4.3 吸顶规则

工具栏 `position: sticky; top: 0`，背景 `var(--bg)` + 滚动后出现 1px 底边框（`--border-light`）。页头不吸顶。

## §5 组件规范

### §5.1 笔记卡片（样板组件，文件卡日后照此平移）

**解剖（自上而下）：**

```
┌─────────────────────── 容器：白底 · 边框1 · 圆角10 · 内边距16 ┐
│ 📌(14px 品牌蓝，仅置顶) 标题 15/600 · 最多2行截断             │ gap 8
│ 摘要 13/行高20 · 最多3行 · 取值链：AI摘要→正文节选→整行隐藏    │ gap 12
│ [#人工] [#人工] [╌AI⋯] [+2]                                  │
│                                                             │ margin-top:auto
│ 3 小时前 · 被 2 篇引用              （12 / sans / muted）    │ ← 无分隔线，纯间距分层
└─────────────────────────────────────────────────────────────┘
悬停：右上浮出 [✎][🗑] 28px 按钮条（L2 投影 · 纯实底 var(--bg-surface) · 禁止半透+模糊）
```

**六态表（缺一不算完成）：**

| 状态 | 视觉 | 触发 |
|---|---|---|
| 默认 | L0 | — |
| 悬停 | `translateY(-2px)` · 边框 `--border-strong` · `--shadow-sm` · 操作条 120ms 淡入 | 指针 |
| 按下 | `scale(0.99)` 120ms | pointerdown |
| 聚焦 | 焦点环（§3.3） | Tab |
| 选中 | 边框 `--primary` · 底色 `--primary-lighter` · 左上勾选框 | 多选模式 |
| 加载 | 骨架卡：三段灰条 + 1.4s 微光，高度与真卡一致（防跳动） | 拉取中 |

**无障碍**：容器 `role="button" tabindex="0" aria-label="打开笔记：{标题}"`；Enter/Space 打开；方向键在卡片间移动。

**反例（现状即反例）**：`padding-right:72px` 给悬停按钮让位（`app.css:2046`）；卡片内 `border-top` 分隔（`app.css:2096`）；meta 用 mono 11px（`app.css:2107`）。

### §5.2 搜索框

宽 232px，聚焦展至 320px（180ms）；高 32px；前置放大镜 16px `--text-muted`；有值时尾部出现清除 ×；快捷键 `/` 聚焦（与设置页一致）；输入防抖 80ms；**输入法组词期（`isComposing`）不过滤**；搜索纯前端、零请求、搜索词不落盘（零痕迹 DNA）。

### §5.3 标签胶囊（筛选栏用）

高 28px 胶囊；静置：`--bg-sunken` 底 + `--text-secondary`；激活：`--primary-light` 底 + `--primary` 文字 + 对勾；可带计数角标（`--text-badge`）。卡片内的标签胶囊为只读变体：高 24px。

### §5.4 排序菜单

触发钮「最近修改 ▾」；菜单 L2、圆角 8、项高 32、单项带 radio 对勾；选项：最近修改 / 最近创建 / 标题；选择记忆于本地偏好。

### §5.5 分段开关（视图切换）

网格 ⊞ / 列表 ☰ 两态；滑块底色 `--bg-surface-active`；切换 160ms；偏好记忆。

### §5.6 空状态三态（文案与动作互不通用）

| 类型 | 图标底色 | 文案 | 动作 |
|---|---|---|---|
| 空库 | `--primary-light` | 「还没有笔记」+ 双向链接玩法说明 | 主按钮「写第一篇笔记」 |
| 筛选无果 | `--bg-sunken` | 「没有找到和 “xx” 匹配的内容」 | 次按钮「清除筛选」 |
| 加载失败 | `--danger-light` | 「笔记列表加载失败」+ 检查网络提示 | 主按钮「重试」 |

> 加载失败渲染成「还没有笔记」是本规范立项的动因之一（`app.js:1097`），三态混淆按事故处理。

### §5.7 编辑器壳层（全屏覆盖层 = L3）

```
┌─ 顶栏 48px ───────────────────────────────────────────────┐
│ ‹(移动端)  笔记 / 周末徒步计划(14/600)      [状态徽标]  [⋯]   │
├─ 内容：max-width 720 居中 ────────────────────────────────┤
│ 标题输入 20/600（无边框）                                    │
│ 正文 15/行高1.7                                             │
├─ 底栏 ──────────────────────────────────────────────────┤
│ 标签编辑 · 421 字 · 约 2 分钟                [取消] [保存]   │
└────────────────────────────────────────────────────────────┘
```

**保存状态徽标四态（§3.2 修正的落点）：**

| 态 | 视觉 | 触发 |
|---|---|---|
| 已保存 | 中性灰 + ✓ + 「已保存 14:32」 | 保存完成 |
| 保存中 | 旋转环 + 「保存中…」 | 请求在途 |
| 未保存 | `--warning-light` 底 + 橙点 + 「未保存」 | 有 dirty 改动 |
| 失败 | `--danger-light` 底 + 「保存失败 · 点击重试」 | 请求失败 |

**溢出菜单**：⋯ 内含 置顶 / 导出 / 移动 / **删除**；删除项仅**菜单内悬停**时文字与底色转 danger（§3.1 危险色规则）。

**反例**：顶栏「编辑笔记」强制 uppercase + 0.06em 字距（`app.css:2272`）；删除与目录/置顶同权重排在顶栏（`app.js:1235`）。

### §5.8 Toast

L2，底部居中，自动 1.6s 消失，成功/失败带语义图标。

## §6 交互规范

### §6.1 时序表

| 行为 | 时长 | 曲线 | 约束 |
|---|---|---|---|
| 按钮按下 | 120ms | ease-out | 仅 transform，不动画布局属性 |
| 卡片悬停起落 | 160ms | cubic-bezier(.2,0,0,1) | 边框+投影+位移同进同出 |
| 操作条/清除钮显隐 | 120ms | ease-out | 仅 opacity |
| 菜单弹出 | 160ms | ease-out | scale(.98→1) + opacity，锚定触发钮 |
| 编辑器开合 | 240ms | ease-out | opacity + translateY(8px→0) |

### §6.2 键盘层

| 按键 | 行为 |
|---|---|
| `/` | 聚焦搜索框 |
| `n` | 新建笔记（焦点不在输入框时） |
| 方向键 | 卡片间移动焦点 |
| `Enter` / `Del` | 打开 / 删除（带确认） |
| `Ctrl+S` | 保存 |
| `Esc` | 关编辑器 → 清搜索 → 关菜单，层级互斥、**一次只关一层** |
| `Ctrl+A` | 多选模式下全选 |

### §6.3 焦点规则

所有可交互元素必须 `:focus-visible` 焦点环；弹层关闭后焦点归还触发元素（沿用 `openModal` 现有机制）。

## §7 功能基线（标准倒逼功能——达标线，不是可选项）

| 标准要求 | 改造 | 级别 |

| 相对时间 | 「3 小时前」，>7 天回落日期 | P0 |
| 批量操作 | 卡片勾选 + 顶部浮出批量条（置顶/改标签/移动/删除） | P1 |
| 视图切换 | 网格/列表分段开关 | P1 |
