# 用户端文件助手模块优化方案

## 0. 需求理解与重构

### 我的理解
用户希望对随行档用户端的文件助手模块进行全面优化，重点是 UI 结构样式，需对齐落地页（landing page）已展示的设计规范。核心参考元素包括：
- AI 对话气泡样式（用户消息灰底 / AI 消息浅蓝底 / 关联文件 chip）
- 语义搜索结果卡片 + 高亮命中片段 + 相关度分数药丸
- 档案室分组侧栏标题 + 分组 + 蓝底高亮激活态

### 真实 JTBD（Job To Be Done）
作为用户，我希望：
1. **快速定位文件**：用自然语言提问，AI 直接给出答案和关联文件，无需手动翻找
2. **清晰理解检索结果**：一眼能看到哪份文件最相关、哪段内容最匹配
3. **高效管理文件分组**：侧栏分组直观清晰，切换流畅
4. **感受产品品质一致**：应用内页与落地页设计语言统一，体验连贯

### 关键假设清单
| 假设 | 优先级 | 验证方式 |
|------|--------|----------|
| 用户主要通过语义搜索而非目录树找文件 | P0 | 埋点统计搜索 vs 导航行为占比 |
| 用户更信任带来源文件的 AI 回答 | P0 | 用户调研 + 点击来源文件的转化率 |
| 当前侧栏分组的视觉层级不够突出 | P1 | 对比测试新旧分组的点击效率 |
| 高亮片段有助于判断相关性 | P1 | A/B 测试有无高亮的结果点击率 |

## 1. 用户与场景

### 目标用户画像
| 用户类型 | 特征 | 核心需求 |
|----------|------|----------|
| **普通用户** | 日常使用浏览器访问，存合同/笔记/文档 | 快速找到、安全预览、AI 问答 |
| **重度用户** | 同时使用守护进程同步，文件量大 | 分组清晰、检索准确、批量操作 |
| **新用户** | 刚注册，文件少 | 引导清晰、理解产品价值 |

### 核心用户故事（User Story）
作为公司办公的用户，我希望输入自然语言问题后，AI 直接给出答案并展示关联文件，这样我不用自己翻阅多个目录就能快速获取信息。

作为家里同步文件的用户，我希望侧栏分组清晰可见，当前所在位置有明显高亮提示，这样我能快速知道自己在哪、切换去哪。

作为语义搜索的用户，我希望搜索结果能展示命中片段高亮，并给相关度打分，这样我能一眼判断哪份文件最相关。

### 场景矩阵

#### 核心场景：AI 对话问文件
| 触发条件 | 预期行为 | 体验要点 |
|----------|----------|----------|
| 用户在聊天框输入问题 | AI 调用工具检索文件，流式回复答案 | 回复自然流畅，工具调用过程透明 |
| 答案关联到具体文件 | 消息下方展示文件来源 chip | chip 带文件图标、文件名，可点击跳转 |
| 用户消息发出 | 消息以灰色气泡展示在右侧 | 气泡圆角柔和，间距适中 |
| AI 回复开始 | 浅蓝气泡展示在左侧，打字动画 | 气泡背景与 landing 一致，视觉连贯 |

#### 核心场景：语义搜索结果
| 触发条件 | 预期行为 | 体验要点 |
|----------|----------|----------|
| 用户输入搜索词并回车 | 展示卡片式结果列表 | 卡片有文件图标、名称、修改时间、大小 |
| 找到匹配内容 | 片段中关键词高亮显示 | 高亮色与品牌蓝协调，不刺眼 |
| 计算出相关度 | 右上角显示相关度分数药丸 | 0.00-1.00 格式，浅色背景，分数醒目 |
| 用户点击结果 | 打开文件预览或进入笔记 | 点击反馈清晰，跳转流畅 |

#### 边缘场景：分组过多滚动
| 触发条件 | 预期行为 | 体验要点 |
|----------|----------|----------|
| 用户分组超过 10 个 | 侧栏可滚动，当前激活分组保持可见 | 滚动条纤细，不破坏整体视觉 |
| 分组名过长 | 文字截断显示，hover 展示全名 | 截断自然，tooltip 位置合适 |

#### 异常场景：无搜索结果
| 触发条件 | 预期行为 | 体验要点 |
|----------|----------|----------|
| 无匹配文件 | 展示友好空状态，建议尝试其他关键词 | 插画风格与 landing 一致，文字有帮助性 |

### 关键体验触点与情感曲线
1. **进入应用** → 侧栏展示分组，默认选中"全部文件" → 安心感：知道自己在哪
2. **输入问题** → 气泡即时发出，打字指示器启动 → 确定感：系统听到了
3. **AI 回复带文件** → 答案清晰，文件 chip 明确 → 信任感：答案有依据
4. **搜索结果** → 相关度排序，高亮片段 → 效率感：一眼找到最相关的

## 2. AI 产品专项（本项目是 AI 产品，必填）

### 模型选型与降级策略
复用 `llm_service.py` 多模型能力：
| 层级 | 模型 | 场景 | 降级策略 |
|------|------|------|----------|
| 主模型 | 用户分配的模型（如 DeepSeek） | 正常对话、工具调用 | 无响应时回退到默认模型 |
| 降级模型 | 系统默认模型 | 主模型不可用时 | 若仍失效，提示"服务暂时不可用" |
| 离线 | - | 完全无 AI 时 | 仅保留纯文件检索功能 |

### Prompt / Agent 设计（落地 brain.py / tools.py）
当前 Agent 已有工具集，优化重点：
- **回答后追加来源标注**：在最终答案末用 `[来源: filename1, filename2]` 格式标注
- **引用位置可视化**：工具返回结果时包含 `snippet` 字段，前端用于高亮展示
- **相关度分数透传**：Chroma 返回的分数透传到前端，用于药丸展示

### RAG 召回-排序策略（indexer.py + Chroma）
| 阶段 | 当前 | 优化方向 |
|------|------|----------|
| 召回 | Chroma 向量检索 | 保持不变，分数透传前端 |
| 排序 | 默认按分数 | 前端可选择按"相关度"或"修改时间"排序 |
| 展示 | 纯列表 | 增加高亮片段 + 相关度药丸 |

### 幻觉/越权/安全防护（guard.py / mask.py / sensitive_paths.py）
- **脱敏保持不变**：AI 回复中的 PII 继续由 `mask.py` 遮罩
- **越权防护保持**：用户只能看到自己的文件
- **新增来源标注验证**：确保标注的文件确实是检索到的，防止幻觉文件名

### 成本/延迟/吞吐预估
- **成本**：新增 UI 不增加 token 消耗，成本持平
- **延迟**：无额外 AI 调用，延迟持平
- **吞吐**：纯前端优化，无后端瓶颈影响

### 评测方案（对接 PROMPTFOO_GUIDE.md）
新增评测维度：
| 维度 | 指标 | 目标值 |
|------|------|--------|
| 来源标注准确率 | 标注文件与实际检索文件一致率 | 100% |
| 相关度展示一致性 | 前端显示分数与 Chroma 返回分数一致率 | 100% |

### 数据回流飞轮
埋点采集：
- 用户点击来源文件 chip 的次数
- 搜索结果点击率与相关度分数的相关性
- 分组切换频次分布

## 5. 交互与体验

### 关键流程线框（文字描述）

#### AI 对话流程
1. 用户输入问题→回车
2. 右侧出现灰色气泡（圆角 10px，padding 12px 16px）展示问题
3. 左侧出现浅蓝气泡占位，显示"正在思考…"+打字动画
4. AI 回复到达后，浅蓝气泡展示完整内容
5. 气泡下方展示文件来源 chips：每个 chip 带文件图标+文件名，可点击
6. 点击 chip 跳转到对应文件预览

#### 语义搜索流程
1. 用户点击侧栏搜索框，输入关键词→回车
2. 主区域展示卡片式搜索结果
3. 每张卡片：左侧文件图标，中间上文件名+修改时间，中间下高亮 snippet，右侧相关度药丸
4. 用户可点击右上角切换"按相关度"或"按时间"排序

#### 分组导航流程
1. 侧栏展示分组列表："全部文件"（蓝底高亮）、"工作"、"证件"、"笔记"…
2. 用户点击"工作"→"工作"变蓝底高亮，"全部文件"恢复常态
3. 主区域展示"工作"分组文件

### 遵循设计系统
- 品牌蓝：#3370FF
- 字体：Geist 无衬线
- 圆角：药丸 999px、气泡 10px、卡片 10px
- 阴影：分层柔和阴影
- 复古宋体已否决：继续使用 Geist + 系统 CJK 字体

### CSP 约束落地
- 事件委托：继续使用 `data-action`，无内联 onclick
- 改 JS/CSS 必升版本号

## 14. 开放问题决策

| 问题 | 决策 | 理由 |
|------|------|------|
| 文件来源 chip 点击后跳转 | 直接预览 | 用户想看文件内容，预览更直接 |
