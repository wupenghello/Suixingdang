# 随行档 · 笔记系统重构 PRD v2.0

## 0. 需求理解与重构

**我的理解**：当前笔记系统功能复杂、视觉设计落后，需要全面优化，与落地页的现代 SaaS 设计风格对齐，实现简洁、美观、易用的笔记体验。

**真实 JTBD**：
- 「我想快速记录想法，不用被复杂操作打断思路」
- 「我希望笔记看起来专业、舒服，愿意反复回看」
- 「我需要笔记与文件统一在一个档案室，双向链接串联知识」

**关键假设**：
- 保持现有数据模型不变（File 表复用）
- 保持 [[wiki]] 双向链接功能
- 优化样式必须提升 ?v 版本号
- 严格遵循 CSP 约束（无内联 onclick）

## 2. AI 产品专项（本项目是 AI 产品，必填）

### 模型选型与降级策略
- **主模型**：用户分配的默认模型（复用现有 `llm_service.py`）
- **降级策略**：
  - AI 不可用时隐藏「AI 整理」按钮
  - 失败时 Toast 提示「AI 暂不可用，请稍后再试」
  - 不阻塞核心笔记编辑功能

### Prompt / Agent 设计
落在现有 `/api/files/ai-enhance` 接口，优化 Prompt 为：

```
你是笔记整理专家，请阅读以下笔记内容，输出：
1. 一句话摘要（不超过 80 字，提炼核心观点）
2. 3-5 个标签（每个 2-6 字，反映主题/类型/状态）
严格按 JSON 格式返回：{"summary":"...","tags":["..."]}

笔记内容：
{content}
```

### RAG 召回—排序策略
- 复用现有 `indexer.py` 语义检索
- 笔记优先展示于搜索结果（可配置权重）

### 幻觉 / 越权 / 安全防护
- 复用 `mask.py` 进行 PII 遮罩
- 严格 `owner_id` 隔离
- 审计日志记录 AI 整理操作

### 成本 · 延迟 · 吞吐预估
- **成本**：每次整理约 500 tokens（可接受）
- **延迟**：控制在 3s 内（超时提示）
- **吞吐**：用户并发极低，无需优化

### 评测方案（对接 PROMPTFOO_GUIDE.md）
新增评测用例：
```yaml
suite: 笔记 AI 整理
cases:
  - description: 技术笔记
    input: "今天调研了 FastAPI，发现异步路由 + Depends 注入很适合我们的场景..."
    expected:
      summary_contain: ["FastAPI", "异步", "注入"]
      tags_contain: ["技术", "调研"]
```

### 数据回流飞轮
- 记录用户对 AI 标签的接受/修改行为
- 可用于未来个性化标签推荐

## 7. 安全与隐私（项目核心，必深挖）

### 脱敏遮罩（mask.py）
- AI 整理返回的摘要自动经过脱敏
- 笔记预览与编辑中的 PII 不渲染（保持原样）

### 路径泄露防护
- 对外使用 `file_id` 而非真实路径（现有机制）
- 编辑器内不暴露物理路径

### 权限校验（guard.py / security.py）
- 所有操作校验 `owner_id`（现有机制）
- AI 整理检查用户 AI 权限开关

### 审计日志（管理端可查）
记录以下操作：
- `note_create` - 新建笔记
- `note_edit` - 编辑笔记
- `note_ai_enhance` - AI 整理
- `note_delete` - 删除笔记

### 多账户隔离边界与越权测试用例

| 测试项 | 预期 |
|-----|-----|
| 用户 A 笔记用户 B 不可见 | 404 Not Found |
